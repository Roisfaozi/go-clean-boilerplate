# Project Master Guide: Go Clean + Next.js Admin

Welcome to the **Casbin DB Project**, an enterprise-grade boilerplate designed for high-performance, secure, and scalable modular REST APIs integrated with a modern Next.js frontend.

---

## 1. Vision & Architecture

This project is built on the foundation of **Clean Architecture**, ensuring that the business logic remains independent of external frameworks, databases, and UI layers.

### The Stack

- **Backend**: Go (Gin, GORM, Casbin, Redis, Asynq, OpenTelemetry).
- **Frontend**: Next.js 14 (App Router, Tailwind CSS, Shadcn UI).
- **Infrastructure**: Docker Compose (MySQL 8, Redis 7, Jaeger, RustFS/MinIO).

### Core Philosophy

1. **Modularity**: Every feature (Auth, User, Org) is a self-contained module.
2. **Security First**: RBAC via Casbin at both API and Middleware levels.
3. **Observability**: Built-in Tracing (OTEL) and Metrics (Prometheus).
4. **Real-time Ready**: Support for distributed WebSockets and SSE.

## 2. Key Modules & Workflows

### 📂 Organization & Multi-Tenancy

The system uses **Organization-based isolation**.

- Users sign up or are invited to Organizations.
- All resource requests (Projects, Users, Audits) must carry an `X-Organization-ID` header.
- The `tenant_middleware` ensures that no data leaks between organizations.

### 🔐 Advanced RBAC (Casbin)

Unlike simple role strings, we use **Casbin** for dynamic, database-backed policies.

- **Roles**: `role:admin`, `role:user`, etc.
- **Domains**: Each organization acts as a Casbin `domain`.
- **Inheritance**: `role:superadmin` can inherit all permissions from sub-roles.

### 📁 Unified Storage & TUS

File uploads are handled via a pluggable **Storage Provider**.

- **Protocols**: Supports standard Multipart for small files and **Tus** for resumable, large file uploads (e.g., Avatars).
- **Backends**: Seamlessly switch between Local Disk and S3 (MinIO/R2).

### 📊 Real-time Presence

Track who is online in real-time.

- **Heartbeat**: Frontend sends pings to keep the session alive.
- **SSE/WS**: Real-time broadcasts for `join`/`leave` events.

## 3. Development Workflow

### Backend (Go)

1. **Update Schema**: Add `.sql` in `db/migrations`.
2. **Run Migrations**: `make migrate-up`.
3. **Seed Data**: `make seed-up`.
4. **Hot Reload**: Run `air` in the root directory.

### Frontend (Next.js)

1. **Navigate**: `cd web`.
2. **Install**: `pnpm install`.
3. **Run**: `pnpm dev`.
4. **API Client**: Use the `api` singleton in `src/lib/api/client.ts`. It handles headers and token refresh automatically.

## 4. Operational Excellence

### 📈 Monitoring

- **Metrics**: Available at `/metrics` (Prometheus).
- **Tracing**: Visualized in **Jaeger** UI (usually port 16686).
- **Audit Logs**: Every critical operation (Create, Update, Delete) is logged asynchronously via **Asynq workers**.

### 🧪 Testing

We maintain high quality via:

- **Unit Tests**: `make test-unit`.
- **Integration**: `make test-integration` (uses real DB in Docker).
- **E2E**: `make test-e2e` (tests full HTTP flow).
